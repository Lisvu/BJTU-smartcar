# 交通标志 YOLO 训练任务(独立工作区)

> 本文件是一个**自包含的任务交接单**。在一个**独立工作区**执行,不依赖也不修改现有的 `ROS2-Foxy/` 车端代码。
> 产出物只有一个:一个微调好的权重 `best.pt` + 对应类别名。训练完成后按「§7 部署回车」交给车端。

---

## 1. 目标

微调一个 YOLOv5s 检测器,识别**打印在 A4 纸上、贴在椅子腿/地面/墙角的黑白交通标志**。
识别结果最终喂给车端 `fusion_node.py` 做决策(看到左转→左转、禁行→停等)。这是课程第 3 项「AI 边缘推理」的加分项。

**关键约束:目标是识别"打印出来的黑白标志纸",不是路边真标志。** 一切数据策略围绕这个域。

---

## 2. 重训还是微调 —— 微调

- **方式:迁移学习,从 `yolov5s.pt`(COCO 预训练)微调。不从零重训。**
- 理由:定制集小(合成 + 少量真图,数千张量级)、类别少(5 类)、算力有限。从零训要海量数据 + 长训练,效果反而差。微调 ~50–100 epoch 即收敛。
- 主干可选冻结前几层(`--freeze 10`)加速小数据训练;数据够时不冻也行。

---

## 3. 类别(默认 5 类,可改)

```
0 turn_left    左转
1 turn_right   右转
2 ahead        直行
3 stop         停车
4 no_entry     禁止通行
```
> 增删类别就改这里 + §5 的 `nc`/`names` + 准备对应模板。类别数变了要重新生成合成集。

---

## 4. 数据策略(全合成:干净矢量模板 → 灰度 → 自造脏数据)

**不存在、也不需要"黑白打印标志贴凳腿"的现成数据集。** Kaggle 上没有,GTSRB 那种路边实拍集又糊又是彩照,跟你的黑白打印域不匹配——**不要用**。正解:找几张干净矢量模板,靠脚本把脏数据造出来。

### 4.1 干净模板 (raw_data)
- 每类 **1 张高清矢量标志(SVG)**,共 4–5 张。来源:**Wikimedia Commons**(官方交通标志,公共域,SVG 无限清晰,免费)。
- 推荐统一用爱尔兰(IE)蓝底白箭头圆牌(风格一致、公共域):
  - 左转 `File:Mandatory_road_sign_turn_left_ahead.svg`
  - 右转 `File:Mandatory_road_sign_turn_right_ahead.svg`
  - 禁止通行 `File:Mandatory_road_sign_no_entry.svg`
  - 直行 / 停车:同系列分类页 `Category:SVG mandatory road signs` 里翻;stop 单独抓一张红八角 "Stop sign" SVG
  - 想用国标就去 `Category:Diagrams of road signs of China`
  - ⚠ 别用新加坡那套(CC BY 4.0 要署名)
- SVG → 用大尺寸渲染成高清 PNG(后面缩小不糊),留透明底(alpha),再进合成管线。

### 4.2 合成数据 = 复刻「打印 → 贴 → 拍」这条物理退化链(自动带标注)

写 `gen_synthetic.py`,对每张模板**按真实物理顺序逐级加工**,最后自动算 YOLO 框。**顺序别打乱**(打印退化必须在透视/光照之前,采集噪声必须在最后):

**阶段 A — 先黑白(模拟"打印成一张纸")**
1. SVG → 高清 PNG。
2. **转灰度**;再降调:黑白打印其实是"黑 + 白 + 少量灰阶",用轻度阈值化 / 减少色阶,更像激光/喷墨打印件。
3. **打印质感**:叠一点纸张纹理、轻微墨点抖动(dither)、对比度略降、极轻高斯模糊(打印机分辨率有限)。到这一步它就"是一张打印出来的纸",不是屏幕上的完美矢量。

**阶段 B — 贴到地上/凳子腿(这张纸变成场景里的一个物体)**
4. **透视扭曲(单应变换)**:纸贴地面/凳腿,相机斜着看——**这条链里最关键的一步**,四角随机偏移。
5. 平面旋转 ±25°(贴歪/翘边)+ 随机缩放(标志占画面 5%–40%,覆盖远近)。
6. 轻微非平面(纸翘角/褶皱,可选):局部扭曲。
7. **光照**:教室顶灯不均匀照明、投影阴影、局部高光/反光(哑光纸也有点)、整体亮度随机(亮/暗环境都要有)。
8. **贴到真实背景**:地板、凳腿、墙角的实拍照片(拍几张教室地面/家具即可当背景库);允许周边物体**部分遮挡**标志一角。

**阶段 C — 车载相机拍下来(采集退化,放最后)**
9. 运动模糊(车在动)+ 散焦模糊(对不准)。
10. 传感器噪声(高斯/ISO)、白平衡偏移、低光。
11. JPEG 压缩伪影。
12. **最终整图保持灰度**(3 通道复制),匹配车端相机看到的黑白纸。

**出标注**:框来自阶段 B 透视变换后标志四角的外接矩形 → 归一化 `class cx cy w h`。
**构图**:每图贴 1–3 个标志、互不重叠;背景多样化。
**幅度=域随机化**:每一级都**随机幅度**——有的近有的远、有的清楚有的糊、有的亮有的暗,模型才不会只认某一种打印/角度/光照。**幅度对准你真车相机的实际水平,别夸张到失真。**
**数量**:训练 ~3000–5000、合成验证 ~500。

### 4.3 现实验证(最低成本,不用标注)
训练数据 100% 脚本合成、零拍摄。但**上车前必须做一次现实检验**,否则测不出 sim-to-real 落差、可能演示翻车。分两档: **最低(必做,零标注)**:打印好 A4 贴凳腿/地面,把车开起来、相机怼着它,**肉眼看认不认、稳不稳**。5 分钟。*加分(可选)**:补拍 20–40 张真实照片、手工标注,当 test 集算真实 mAP(§7)。**这两档都不进训练。**
